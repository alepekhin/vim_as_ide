# snake-game

## Обзор проекта

Это десктопная игра "Змейка", написанная на Java с использованием JavaFX для графического интерфейса.
Игрок управляет змейкой с помощью клавиш-стрелок, собирает еду (красные квадраты) и избегает столкновений
со стенами и собственным хвостом. После поражения можно перезапустить игру, нажав `Пробел`.

## Технологический стек

- **Язык**: Java 25
- **Сборка**: Apache Maven 3.x
- **GUI-фреймворк**: JavaFX 18 (`javafx-controls`, `javafx-fxml`)
- **Группа/артефакт**: `com.example:snake-game:1.0-SNAPSHOT`

## Структура проекта

```
src/main/java/
├── Main.java              # Точка входа: запускает JavaFX-приложение
└── snake/
    ├── SnakeGame.java     # Основной класс Application, игровой цикл, обработка клавиш
    ├── Board.java         # Игровое поле: сетка из прямоугольников 15×22
    ├── Snake.java         # Логика змейки: движение, столкновения, тело (Deque<Point>)
    └── Food.java          # Еда: случайное размещение на поле
```

### Описание модулей

- **`Main`** — статическая точка входа без аргументов. Вызывает `Application.launch(SnakeGame.class)`.
- **`SnakeGame`** — главный класс приложения JavaFX. Наследует `Application`.
  Игровой цикл реализован через `javafx.animation.Timeline` (ранее использовался daemon-поток + `Runnable`).
  Обрабатывает нажатия клавиш, создаёт и пересоздаёт игровые объекты при перезапуске,
  управляет отображением `Game over` через `Label` внутри `StackPane`.
- **`Board`** — независимая от игровой логики доска. Представляет собой сеть `Rectangle[COL_NUM][ROW_NUM]`
  (`COL_NUM = 22`, `ROW_NUM = 15`, размер клетки 30×30 пикселей). Отвечает только за отрисовку фона
  и добавление ячеек в `GridPane`.
- **`Snake`** — змейка. Тело хранится в виде `LinkedList<Point>`. Голова — `getFirst()`, хвост — `getLast()`.
  При столкновении со стеной или самой собой выбрасывает `IllegalStateException` с текстом `"Game over. Press space"`.
- **`Food`** — еда случайного цвета (`Color.RED`), появляется в случайной свободной клетке.
  При создании проверяется, чтобы еда не оказалась на теле змейки.

## Ответственности и API классов

### `Main` (пакет — корневой)

**Ответственность**: статическая точка входа в приложение. Не содержит игровой логики.

| Элемент | Сигнатура | Описание |
|---------|-----------|----------|
| `main` | `public static void main(String[] args)` | Запускает JavaFX-приложение через `Application.launch(SnakeGame.class)`. |

---

### `SnakeGame` (пакет `snake`)

**Ответственность**: главный класс JavaFX `Application`. Управляет UI, игровым циклом, обработкой клавиш, созданием/пересозданием игровых объектов и отображением сообщения о поражении.

| Элемент | Сигнатура | Описание |
|---------|-----------|----------|
| `PAUSE_MS` | `private static final int PAUSE_MS = 200` | Задержка между тактами игрового цикла (мс). |
| `start` | `@Override public void start(Stage primaryStage)` | Инициализирует `StackPane`, `Board`, `Label` для Game Over, `Scene`, обработчик клавиш и запускает первую игру (`initGame()`). |
| `initGame` | `private void initGame()` | Останавливает старый `Timeline`, сбрасывает флаг `isGameOver`, создаёт новые объекты `Snake` и `Food`, отрисовывает начальное состояние и запускает новый `Timeline`. |
| `tick` | `private void tick()` | Игровой такт: вызывает `snake.move()`, обновляет `Food` при поедании, перерисовывает доску. Перехватывает `IllegalStateException` для обработки Game Over.
| `handleKey` | `private void handleKey(KeyEvent event)` | Реакция на нажатие клавиш: стрелки меняют `direction` (с запретом разворота на 180°), `SPACE` перезапускает игру при `isGameOver`. |

**Внутреннее состояние**:
- `board` — игровое поле.
- `snake` — логика змейки.
- `food` — еда.
- `direction` — текущее направление движения (`Direction`).
- `timeline` — JavaFX `Timeline`, реализующий игровой цикл.
- `gameOverLabel` — `Label` для отображения надписи "Game over. Press space".
- `isGameOver` — флаг окончания игры.

---

### `Board` (пакет `snake`)

**Ответственность**: визуальное представление игрового поля. Создаёт сетку `Rectangle`, очищает и перерисовывает её в соответствии с состоянием игры. Не содержит игровой логики.

| Элемент | Сигнатура | Описание |
|---------|-----------|----------|
| `COL_NUM` | `public static final int COL_NUM = 22` | Число столбцов сетки (ширина поля в клетках). |
| `ROW_NUM` | `public static final int ROW_NUM = 15` | Число строк сетки (высота поля в клетках). |
| `CELL_SIZE` | `public static final int CELL_SIZE = 30` | Размер одной клетки в пикселях (30×30). |
| (конструктор) | `public Board()` | Создаёт `GridPane`, заполняет его `Rectangle[COL_NUM][ROW_NUM]` чёрным фоном (`Color.BLACK`) и серой сеткой (`Color.DARKGRAY`). |
| `getGrid` | `public GridPane getGrid()` | Возвращает `GridPane` для добавления в сцену.
| `clear` | `public void clear()` | Сбрасывает цвет всех клеток в `Color.BLACK`.
| `drawSnake` | `public void drawSnake(List<Point> body)` | Рисует тело змейки `Color.GREEN`, голову — `Color.LIME`. Координаты проверяются на вхождение в границы поля.
| `drawFood` | `public void drawFood(Point food)` | Рисует еду цветом `Color.RED`.

**Внутреннее состояние**:
- `cells` — двумерный массив `Rectangle[COL_NUM][ROW_NUM]`.
- `grid` — `GridPane`, содержащий все ячейки.

---

### `Snake` (пакет `snake`)

**Ответственность**: управление состоянием змейки (позиция тела, движение, проверка столкновений). При столкновении со стеной или самой собой выбрасывает исключение.

| Элемент | Сигнатура | Описание |
|---------|-----------|----------|
| (конструктор) | `public Snake(int width, int height)` | Создаёт змейку длиной 3 клетки, расположенную горизонтально от центра поля влево. |
| `getBody` | `public List<Point> getBody()` | Возвращает тело змейки (`LinkedList<Point>`). Голова — `get(0)`, хвост — последний элемент. |
| `getHead` | `public Point getHead()` | Возвращает голову змейки (`body.getFirst()`). |
| `move` | `public boolean move(Direction direction, Point foodPosition)` | Перемещает змейку на одну клетку в заданном направлении. Возвращает `true`, если еда была съедена. Выбрасывает `IllegalStateException("Game over. Press space")` при столкновении со стеной или самой собой. |

**Внутреннее состояние**:
- `body` — `LinkedList<Point>`, хранит координаты сегментов тела.
- `width`, `height` — границы игрового поля для проверки столкновений.

| Метод | Детали реализации |
|-------|------------------|
| `move` | Создаёт новую точку головы на основе старой, добавляет в начало списка. Если еда не съедена — удаляет хвост. Затем проверяет столкновение головы со стеной и с телом. |

---

### `Food` (пакет `snake`)

**Ответственность**: управление позицией еды на поле. Генерирует случайные координаты, исключая клетки, занятые змейкой.

| Элемент | Сигнатура | Описание |
|---------|-----------|----------|
| (конструктор) | `public Food(int colNum, int rowNum, List<Point> snakeBody)` | Инициализирует размеры поля и сразу вызывает `respawn(snakeBody)` для первого размещения еды. |
| `getPosition` | `public Point getPosition()` | Возвращает текущую позицию еды (`Point`). |
| `respawn` | `public void respawn(List<Point> snakeBody)` | Генерирует новую случайную позицию (`Random`) в пределах `[0, colNum)` и `[0, rowNum)` до тех пор, пока она не окажется свободной от тела змейки. |

**Внутреннее состояние**:
- `colNum`, `rowNum` — размеры игрового поля.
- `random` — экземпляр `Random`.
- `position` — текущая позиция еды.

---

### `Direction` (пакет `snake`)

**Ответственность**: перечисление направлений движения змейки.

| Константа | Описание |
|-----------|----------|
| `UP` | Вверх (уменьшение `y`). |
| `DOWN` | Вниз (увеличение `y`). |
| `LEFT` | Влево (уменьшение `x`). |
| `RIGHT` | Вправо (увеличение `x`). |

---

### `Point` (пакет `snake`)

**Ответственность**: immutable-подобная структура для хранения целочисленных координат `(x, y)` на игровом поле. Поддерживает сравнение по значению.

| Элемент | Сигнатура | Описание |
|---------|-----------|----------|
| `x` | `public int x` | Координата X. |
| `y` | `public int y` | Координата Y. |
| (конструктор) | `public Point(int x, int y)` | Создаёт точку с заданными координатами. |
| (конструктор копирования) | `public Point(Point other)` | Создаёт копию существующей точки. |
| `equals` | `@Override public boolean equals(Object obj)` | Сравнение по значению `x` и `y`. |
| `hashCode` | `@Override public int hashCode()` | Хеш-код на основе `Objects.hash(x, y)`. |

## Сборка и запуск

### Требования

- JDK 25
- Apache Maven
- JavaFX модульные библиотеки (если запускается не через `mvn javafx:run`)

### Команды сборки

```bash
# Компиляция и создание fat JAR
mvn package

# Результат: target/snake-game-1.0-SNAPSHOT.jar (включает зависимости через maven-shade-plugin)
```

### Команды запуска

```bash
# 1. Через JavaFX Maven Plugin (рекомендуется)
mvn javafx:run

# 2. Через собранный JAR (требует JavaFX в module path или CLASSPATH)
java -jar target/snake-game-1.0-SNAPSHOT.jar

# 3. Запуск с удалённой отладкой (JDWP на порту 5005)
./run.sh
# Эквивалентно:
# java -agentlib:jdwp=transport=dt_socket,server=y,suspend=n,address=*:5005 -jar target/snake-game-1.0-SNAPSHOT.jar
```

### Управление в игре

| Клавиша | Действие |
|---------|----------|
| `↑` `↓` `←` `→` | Смена направления движения |
| `Пробел` | Перезапуск после Game over |

## Архитектура во время выполнения

- **Единственный поток**: JavaFX Application Thread — и UI, и игровая логика работают
  на этом потоке через `Timeline` (`javafx.animation`). Это исключает гонки данных,
  не требует `Platform.runLater` и упрощает отладку.
- **Игровой цикл**: `Timeline` с `KeyFrame` на `PAUSE_MS = 200` мс, цикл бесконечный (`INDEFINITE`).
  На каждом такте вызывается `tick()`, который перемещает змейку, проверяет столкновения
  и обновляет доску.
- **Перезапуск**: при нажатии `SPACE` после Game Over вызывается `initGame()`,
  который останавливает старый `Timeline`, сбрасывает состояние и создаёт новый `Timeline`.
- **Обработка ошибок**: исключения `IllegalStateException` из `Snake.move()` перехватываются
  напрямую в `tick()` — UI-обновление происходит на том же JavaFX-потоке, дополнительная
  синхронизация не требуется.

## Стиль кода и соглашения

- Комментарии и JavaDoc в исходниках пишутся на русском языке.
- Используются стандартные соглашения Java (camelCase для методов/переменных, PascalCase для классов).
- Константы объявляются как `public static final`.
- В проекте встречаются опечатки в названиях методов (`drow` вместо `draw`, `WIFTH` вместо `WIDTH`).
  Эти имена сохранены для обратной совместимости вызовов между классами.
